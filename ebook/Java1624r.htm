Preface

This chapter contains review questions and answers keyed to the chapter titled Java1624: Array Objects, Part 2.

The questions and the answers are connected by hyperlinks to make it easy for you to navigate from the question to the answer and back again.

Questions

Question 1.

True or False? When declaring a reference variable capable of referring to an array object, the array type is declared by writing the name of an element type followed by some number of empty pairs of square brackets [].

Answer 1

Question 2

True or False? The components in an array object may refer to other array objects. The number of bracket pairs used in the declaration of the reference variable indicates the depth of array nesting (in the sense that array elements can refer to other array objects).

Answer 2

Question 3

True or False? An array's length is part of its type or reference variable declaration.

Answer 3

Question 4

True or False? Multi-dimensional arrays are not required to represent rectangles, cubes, etc. They can be ragged.

Answer 4

Question 5

True or False? The normal rules of type conversion and assignment compatibility apply when creating and populating array objects.

Answer 5

Question 6

True or False? Object is the superclass of all other classes. Therefore, an array of element type Object is capable of storing references to objects instantiated from any other class. The type declaration for such an array object would be Object[].

Answer 6

Question 7

True or False? An array of element type Object is not capable of storing references to other array objects.

Answer 7

Question 8

True or False? If the declared element type for the array object is one of the primitive types, the elements of the array can be used to store values of any primitive type that is assignment compatible with the declared type (without the requirement for a cast).

Answer 8

Question 9

True or False? If the declared element type is the name of a class, (which may or may not be abstract), a null reference or a reference to any object instantiated from the class or any subclass of the class may be stored in the array element.

Answer 9

Question 10

True or False? If the declared element type is an interface type, a null reference or a reference to any object instantiated from any class that implements the interface can be stored in the array element.

Answer 10

Question 11

True or False? A reference variable whose declared type is an array type contains space for the array components.

Answer 11

Question 12

True or False? A reference to an array object can refer to different array objects (of the same element type and different lengths) at different points in the execution of a program.

Answer 12

Question 13

True or False? When declaring an array reference variable, the square brackets [] must appear following the variable name.

Answer 13

Question 14

True or False? Once an array object is created, its type and length never changes.

Answer 14

Question 15

True or False? The length of the array (number of elements in the array) is always available as a final instance variable named length.

Answer 15

Question 16

True or False? An array element is accessed by an expression whose value is an array reference followed by an indexing expression enclosed by matching square brackets.

Answer 16

Question 17

True or False? If an attempt is made to access the array with an invalid index value, a non-fatal warning is displayed.

Answer 17

Question 18

True or False? If the elements in an array are not purposely initialized when the array is created, the array elements will be automatically initialized with default values.

Answer 18

Question 19

True or False? The values in the array elements may be purposely initialized when the array object is created using a comma-separated list of expressions enclosed by matching curly brackets.

Answer 19

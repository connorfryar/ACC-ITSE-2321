Preface

This chapter contains review questions and answers keyed to the chapter titled Java1627: Static Members.

The questions and the answers are connected by hyperlinks to make it easy for you to navigate from the question to the answer and back again.

Questions

Question 1.

True or False? The existence of static members tends to break up or complicate the simple OOP structures that I have discussed in previous chapters in this eBook.

Answer 1

Question 2

True or False? The overuse of static members can lead to problems similar to those experienced in languages like C and C++ that support global variables and global functions.

Answer 2

Question 3

True or False? The Java class library contains a class named Object and another class named Class.

Answer 3

Question 4

True or False? Instance variables and instance methods are declared static.

Answer 4

Question 5

True or False? Class variables and class methods are not declared static.

Answer 5

Question 6

True or False? Every object instantiated from a given class shares the same single copy of each instance variable declared in the class definition.

Answer 6

Question 7

True or False? An instance variable or an instance method can only be accessed by using a reference to the object that owns it. Even then, it may or may not be accessible depending on the access modifier assigned by the programmer.

Answer 7

Question 8

True or False? The single shared copy of a class variable or a class method can be accessed in either of two ways:

  • Via a reference to any object instantiated from the class.
  • By simply joining the name of the class to the name of the class variable or the class method.

Again, the variable or method may or may not be accessible, depending on the access modifiers assigned by the programmer.

Answer 8

Question 9

True or False? It is very often appropriate to use final static variables, as constants in your programs. It is rarely appropriate to use non-final static variables in your programs.

Answer 9

Question 10

True or False? It is often appropriate to use static methods in your programs, provided there is no requirement for the method to remember anything from one call to the next. An appropriate example of a static method is the sqrt method of the Math class.

Answer 10

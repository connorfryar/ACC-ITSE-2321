Update: Java programs with two or more source code files

 

Students,

Chapter Jb0110: Programming Fundamentals, Getting Started in the course eBook explains how to write, compile, and run Java programs consisting of a single source code file. The purpose of this announcement is to provide an example showing how to write, compile, and run Java programs consisting of two or more source code files, which is the case for most of the assignments in this course.

One student reported that the instructions given in the original version of this announcement would not work on his computer. As it turned out, that student's computer apparently had an invalid preset classpath installed which was preventing the sample program from compiling. Those instructions have been updated in this version to use the -cp switch to tell the compiler to ignore a preset classpath, valid or otherwise. If you are having difficulty compiling and running the sample program on your system, try using the updated instructions provided below.

Begin by creating a new empty folder named Hello02. Create a Java source code file in that folder named Hello02.java containing the code shown below:

class Hello02{
public static void main(String[] args){
new Hello02Runner().run();
}//end main
}//end class

Create another Java source code file in that folder named Hello02Runner.java containing the code shown below:

class Hello02Runner{
void run(){
System.out.println("Hello World from Hello02");
}//end main
}//end class

Open a command line window in the folder named Hello02 and enter the following command to compile the program. Note that this command uses the -cp switch to tell the compiler to ignore a preset classpath and search only the current folder.

javac -cp .; Hello02.java

If there are no errors, the command prompt should reappear and the following two files should have been created in the folder named Hello02. (You may need to press the F5 key to make them visible.)

Hello02.class
Hello02Runner.class

Next, enter the following command to run the program. Note that this command uses the -cp switch to tell the Java virtual machine to ignore a preset classpath and search only the current folder.

java -cp .; Hello02

The following output should appear on the command-line screen:

Hello World from Hello02

Note that these instructions assume that you set the path environment variable when you installed the Java software. If you didn't, you will need to deal with the path when entering the commands. In that case, your commands would be something like the following depending on the version of the Java software that was installed and where the software was installed on your hard drive. (Note that the quotation marks are required in the commands due to the space character in the folder name: Program Files.)

"C:\Program Files\Java\jdk-13\bin"\javac -cp .; Hello02.java
"C:\Program Files\Java\jdk-13\bin"\java -cp .; Hello02
